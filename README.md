# HSIL-2026

Private repository for our HSIL-2026 project team.

## Working direction

Current working concept (subject to refinement with the team):

A clinician-facing decision-support flow for inherited retinal disease cases that helps reduce interpretation friction around genomic/imaging context and produces a structured clinician-review draft for next actions.

This is **support-first**, not an autonomous diagnostic tool.

## Repository layout

- `00_admin/` – team operations, roles, and sprint notes
- `01_sources/` – problem framing, references, and source materials
- `02_research/` – evidence, competitor scan, and clinical assumptions
- `03_solution/` – proposed architecture, features, and product scope
- `04_demo/` – demo flow, data sets, walkthroughs, and checklist
- `05_pitch/` – pitch deck outline, speaking notes, and FAQ
- `06_deliverables/` – submission-ready artifacts and packaging
- `99_archive/` – retired drafts and historical notes

## Collaboration defaults

- Keep scope tight: one workflow, one strong user moment, clear boundaries.
- Every claim in docs should have a source or a test path.
- Prefer one source of truth per section (README + corresponding doc).
