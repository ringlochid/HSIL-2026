# HSIL-2026 Roadmap

## 1) Team + scope lock (now)
- Confirm core persona and workflow with team.
- Freeze user outcome: who this helps, in one sentence.
- Freeze MVP boundaries (what the app *does not* do).

## 2) Problem and evidence pack (next)
- Convert Slack/group discussions into concrete pain points.
- Collect 8–10 evidence references for the problem statement.
- Define one user path for the first demo.

## 3) Solution design (pre-build)
- Draft data flow: input -> structure -> reasoning summary -> clinician review.
- Define schema for one core room/record type.
- Define fallback behavior if any data field is missing.

## 4) Prototype (build)
- Implement the minimal route-to-decision UI (one primary screen).
- Implement deterministic demo scenario(s) with local/static data.
- Build a review/edit step before final output.

## 5) Demo + pitch prep
- Record scripted 3-minute flow.
- Prepare judge-facing talking points: clinical realism, safety boundary, feasibility.
- Run full dry-runs in mock/hurry conditions.

## 6) Submission polishing (final)
- Clean deck + repo (README, one architecture diagram, one scenario script).
- Prepare “what we built / what we didn’t build” summary.
- Verify all required HSIL deliverables are present.
